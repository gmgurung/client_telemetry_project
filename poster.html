The Problem

Digital user frustration leads to abandonment, but relying on hardcoded rules (e.g., "if 3 clicks, then mad") fails to scale across complex multi-platform flows.

Our Objective: Autonomously detect anomalous behavior via an unsupervised machine learning pipeline.

Methodology

The system utilizes a modern, decoupled ingestion-to-inference architecture:

  • Telemetry SDK: A custom JavaScript payload captures granular events (rage clicks, idle time, DOM refocus, erratic scrolling).
  • Synthetic Generation: Playwright automates 4 user personas (Normal, Frustrated, Lost, Error) over mock UI flows to generate robust training datasets.
  • Feature Engineering: Vectorized Pandas processes NDJSON streams into 13 behavioral features (e.g., avg inter-event gap, unique routes).
  • The Ensemble: A dual-model approach utilizing a TensorFlow Autoencoder and a Scikit-Learn Isolation Forest for robust anomaly detection.

Pipeline Results

The SageMaker endpoint normalizes the ensemble output into a 0-10 Frustration Score, mapped to operator severities:

  • Normal (<7): Standard interactions.
  • Medium (7-9): Elevated UI friction.
  • High (≥9): Anomalous behavior (immediate intervention recommended).

Future Work

  • Stateful Micro-Batching: Migrating from S3 file-polling to direct RDS querying for O(1) read overhead at scale.
  • Semi-Supervised Feedback: Adding Logistic Regression layers to dynamically tune thresholds based on human operator labels.
