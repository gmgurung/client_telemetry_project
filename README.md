# Project Overview
The Vanguard Client Telemetry Project is a three-phase system designed to collect user interaction data, analyze user behavior using machine learning models, and visualize frustration analytics through an administrative dashboard. The goal of the project is to identify potentially frustrating user experiences and provide insights that can be used to improve website usability.

# Features Implemented

# Phase 1
- Telemetry collection framework
- User interaction tracking
- Synthetic user session generation
- Event logging and ingestion

# Phase 2
- Telemetry preprocessing and feature extraction
- Machine learning model training
- Frustration scoring and severity classification
- Automated prediction generation and storage

# Phase 3
- Interactive dashboard for session visualization
- Alerts and queue prioritization views
- Session-level analytics and metrics
- Scenario detection and event inspection

# Installation Requirements
The project requires:
- Node.js and npm
- Python 3.x
- AWS credentials with access to project resources
- Required Python and JavaScript dependencies
Detailed installation instructions are provided in the Code Documentation.

# Running Phase 1
Start the telemetry collection and simulation environment using the Phase 1 application and simulation scripts.

# Running Phase 2
Execute the preprocessing, training, and inference pipeline using the AWS SageMaker environment and configured S3 resources. Execution steps and CLI commands listed on code documentation document. 

# Running Phase 3
Start the backend API service and frontend dashboard application to visualize prediction results and session analytics.
Detailed startup instructions are available in the Code Documentation.

# Known Issues
- Dashboard data depends on the availability of prediction outputs generated by Phase 2.
- Large telemetry datasets may increase dashboard loading times.
- AWS credentials must be configured correctly to access required resources.

# Features Not Implemented
- Real-time streaming analytics
- Production authentication and authorization
- Automated notification system for high-severity sessions
- Production deployment infrastructure
